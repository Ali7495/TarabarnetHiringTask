## 🧠 Domain Layer Explanation

### فولدر Common
مواردی که در سطح لایه مورد استفاده قرار میگیرند را در اینجا ایجاد کردم مانند BaseEntity که شامل prop هایی است که 
کلاس های Entity از آن ها استفاده میکنند.

### فولدر Contact
در واقع با ساخت این فولدر بحث Bounded Context را پیاده سازی کردم. بنابراین تمام Entity های مرتبط با Conatc در این فولدر قرار میگیرند.

### فولدر Event
برای پیاده سازی ثبت اتفاقات در جریان اپلیکیشن

## Test Layer Explanation
لایه دامین رو چند تا تست نوشتم که از نظر منطق بیزینس چک شوند. برای لایه اپلیکیشن هم یک تست نوشتم تا ریپازیتوری رو هم تست کند.
ساختار تست رو بصورت کلی به 3 قسمت دامین، اپلیکیشن و API تقسیم بندی کردم.
به دلیل مشغله و نبود وقت کافی تست مربوط به Api نوشته نشد و فقط تست های کمی نوشته شد.

## Application Layer Explanation

با استفاده از CQRS عملیات ها پیاده سازی شده
بحث Validation هم با استفاده از FluentValidation بوده و با کمک MediateR هندل شدند.
